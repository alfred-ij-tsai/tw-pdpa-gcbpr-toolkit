# 個人資料保護法（PDPA）法令遵循工具箱

**Global CBPR System 對照版（GCBPR Edition）**

[![License: CC BY-NC-SA 4.0](https://img.shields.io/badge/License-CC%20BY--NC--SA%204.0-lightgrey.svg)](https://creativecommons.org/licenses/by-nc-sa/4.0/)

一套以台灣個資法、施行細則、產業別安全維護辦法、行政院要點為核心，涵蓋**日常法令遵循、當事人近用請求（DSAR）、個資事故應對**三大實務場景的法遵工具箱，設計給律師、企業法務、法遵合規人員，以及需要快速掌握個資法遵循義務的使用者。

本工具箱的核心內容，是完整的台灣個資法日常合規應用——每一項法條義務、每一份佐證文件，都直接對應到機構實際運作的蒐集、處理、利用、事故通報等場景。在此基礎上，本版本另外逐項對照 **Global CBPR System Program Requirements**，讓機構在完成國內法遵的同時，也能一併檢視自身是否具備跨境資料流通所需的佐證能力。

搭配 AI 助理使用，可將本工具箱轉化為互動式法遵引導流程。

## 為什麼選擇 Global CBPR？

**Global CBPR System（Global Cross-Border Privacy Rules，全球跨境隱私規則體系）** 是由 Global CBPR Forum 主導的國際隱私驗證框架。該論壇於 2022 年由澳洲、加拿大、日本、南韓、墨西哥、菲律賓、新加坡、**中華臺北**及美國共同成立，宗旨在於推動資料保護與資料自由流通，並發展可跨經濟體互信互認的隱私驗證機制。**台灣是論壇創始會員之一**，由國家發展委員會（國發會）負責協調、資訊工業策進會（資策會）擔任我國當責機構（Accountability Agent），持續參與該體系的推廣與運作。

什麼是計畫要求 (Program Requirements)？ 計畫要求是 GCBPR 認證機制的核心，共 50 題，區分為「告知」「蒐集限制」「個人資料利用」「選擇」「個人資料完整性」「安全維護」「近用及更正」「問責」等8個章節，每題皆搭配當責機構（Accountability Agent）用以查核企業回覆是否合格的評估標準。此工具箱的「GCBPR 對照」，即是將個資法各項義務對應到這 50 題計畫要求中的具體題號，讓機構在準備國內法遵文件時，能一併掌握其內容是否也足以支撐 GCBPR 驗證申請。

可以把 GCBPR 想像成**隱私版的護照**——當一間企業能夠完整回答 GCBPR 問卷中的各項問題，就代表其個資保護作為已具備跨境可辨識、可互信的基礎，而不只是滿足單一法域的最低要求。

具體而言，選擇 GCBPR 的理由包括：

- **與台灣的國際政策方向一致**：GCBPR 是我國政府實際參與、推動的國際隱私框架，而非單純的外部標準，讓機構在落實個資法遵循的同時，也能同步建立起與國際隱私驗證框架接軌的能力。
- **官方文件公開透明**：Enforcement Map 均為公開文件，可自由引用、轉載，適合作為開放授權工具箱的對照依據
- **兼顧國內合規與跨境接軌**：透過 GCBPR 問卷逐項對照個資法條文與施行細則，機構不僅能掌握國內法遵義務，也能同步檢視自身是否具備跨境資料流通所需的隱私保護佐證能力

## 文件架構

| 編號 | 文件名稱 | 用途 |
| --- | --- | --- |
| 00 | 封面目錄 | 合併版封面與目錄 |
| 01 | AI 使用 Prompt | 三路徑工作流程（A 日常法遵 / B DSAR / C 個資事故），驅動 AI 助理互動引導 |
| 02 | 法規盤點對照表 | 依中央目的事業主管機關別，盤點個資法及資安法子法 |
| 03 | 個資生命週期法令遵循主表 | 蒐集→處理→利用→儲存→傳輸→跨境傳輸→銷毀，逐階段法條義務與應備文件，並對照 GCBPR |
| 04 | DSAR 對照表 | 當事人查詢、複製、更正、停止、刪除五項權利之回應義務與期限 |
| 05 | 個資事故對照表 | 依行政院要點六章架構，逐項對照個資安維案件通報、調查、裁罰義務 |

每份文件皆以台灣個資法條文及施行細則為主軸，GCBPR 對照僅作為額外一欄佐證資訊，不影響文件本身作為國內法遵工具的完整性。

## 三種使用情境（路徑）

本工具箱設計為搭配 AI 助理（如 Claude、ChatGPT 等）使用，透過 01 文件中的 Prompt 驅動互動式問答：

- **路徑 A｜日常法令遵循**：建立或檢視個資安全維護計畫、生命週期各階段文件（搭配 02 ＋ 03）
- **路徑 B｜當事人近用請求（DSAR）**：收到當事人查詢/複製/更正/停止/刪除請求時，確認回應義務與 15 日期限（僅需 04）
- **路徑 C｜個資事故應對**：發生或疑似個資外洩、竊取等事故時，確認 24/72 小時通報時限與後續義務（搭配 02 ＋ 05）

## 使用方式

1. 將 `01_AI使用Prompt` 全文，連同對應情境所需文件（02～05）一併提供給 AI 助理
2. 依 AI 助理的提問逐步回答（每次僅需回答一個問題）
3. 取得客製化法遵查核清單或行動方案
